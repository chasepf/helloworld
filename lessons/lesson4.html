Full Stack: Intro to Networks and Allowing Comments

Network Vocab and Terminology

Knowing the vocab of new subject helps facilitate learning. Here are some of them.

Network: A group of entities that can communicate with each other even though the entities may not be connected directly. By definition a network has more than 2 people. Networks are a 3000 year old idea. The internet is an example of a network.

Latency: The time it a message to get from source to destination. Measure din milliseconds.

Bandwidth: The amount of information that can be transmitted per unit of time. Measured in bits per second.

Bit: The smallest unit of information. It decides between two things. Think 'on' or 'off', 'yes' or 'no', '0' or '1'.

Protocols: A set of rules that people agree to, that tell you how two entities talk to eachother.

URL: Uniform Resource Locator

Caching or Cache: Something that stores data so you don't have to retrieve it later. Can be used to make data requests faster.

1. A way to encode and interpret messages

Servers

A server is a computer running a server program. The purpose of a server is to respond to HTTP (a major protocol of the web) requests. There are two types of responses a server will do: STATIC and DYNAMIC.

STATIC responses are made when stactic content (pre-written files such as images) are requested.

DYNAMIC responses are made on the fly by programs on the server. These programs are called web applications. Web Applications live on the server and generate responses based on the request.

Status Codes:

  • 200 = Ok, document found and where it should be
  • 300 = Found, but may need to alter request to find the doc
  • 400 = Not found, browser error
  • 500 = Server Error, something is wrong server side

HTTP: HyperText Transfer Protocol

HTTP is the protocol for the Client-Server model. Clients send HTTP requests using a GET or POST method. A server will then respond with HTTP Response with the content that was requested.

An example of a GET Request: GET /foo HTTP/1.1

An example of a Response would be: HTTP/1.1 200 OK

Always use User-Agents appropriately!!

This is the mantra to follow.

The user agent tells the server who is trying to make contact. Good for identifying spammers and hackers.

GET

  • Parameters in URL
  • Used fetching documents from a server
  • Maximum URL length
  • OK to cache
  • This shouldn't change the serva data, only to retrieve data

POST

  • Parameters in the body
  • Used to update data
  • There isn't a max length
  • Not OK to cache
  • Allowed to change server data

The Importance of Validating Input

Validation is verifying on the server side the we receive is what we expected to receive. This is very important because it controls what information the user is allowed to send to the server. A server can receive junk and it needs to be smart enough to deal with somebody sending broken data. It's important to validate parameters otherwise we can receive junk that can harm the server.

It's also important to have the user's experience in mind when validating info. For example if the users submitted some invalid data through a form (like a typo) we should let them know by displaying an error message so that they can re-submit valid data before it is stored on the database. And to also consider times when a blank form is submitted.

HTML Templates and Abstraction

Templates typically refer to a template library that is used to build complicated strings.

Using templates allows you to keep your code and html seperate but you can access each from one another.

The benefits of templates are:

  • Great way to separate different types of code
  • Makes code more readable
  • More secure websites
  • HTML that is easier to modify
  • Maintaining code editor syntax highlighting
  • Template inheritance
  • Use simple Jinja syntax for repeated HTML elements

Avoiding code repetion is a huge part of good programming. You only want to write code that performs a specific function once. This makes updating your code way easier. Also, when humans do repetitive things we increase our chance of making mistakes. This causes bugs in code. When code repetition is low, there is less chance for bugs and it makes it easier to find bugs when they do occur.

Jinja's allows for code inheritance between html templates. Jinja's built-in loops and decision branching elements help to reduce repetitive code. You can use Jinja loops to wrap various elements in highly repetitive elements like the html div tag structure. For example, Jinja can be applied to wrap the course note lessons and concepts into the highly repeating div tag structure. Then course note elements can be inserted into the Jinja code by using string substitution on the Jinja code with the variable element. This supports easy changes to your layout and reduces the amount of lines of code, which means fewer lines of code to look through when searching for bugs.
