The web and HTML

What is the web?

The web is a giant network of computers that talk to each other using a protocol call HTTP. A webpage is document written in HTML (HyperTextMarkupLanguage). A web browser is a program that interprets and displays the info. The main pieces of the web are: users, the net and servers.

What is coding?

Coding is writing text in a language that a computer can understand. If you know how to talk to a computer, you can tell it do anything! That's pretty neat!!!

Computers ain't dat bright

Computers are very literal. They don't intepret the way a human does. This means one little typo can become a huge problem. This is an example of 'Technological Empathy'.

Inline vs block

Elements are either inline or block. Block elements form an invisible box around them. This is useful for diving your page up into sections for styling.

Examples of Inline elements: a, span, br, img and strong.

Examples of Block elements: p, form and div.

Thinking like a programmer

There are too many details to remember. Focus on the big ideas then know what and how to look up. Google to the rescue!!!

5 different types of thinking:

  1. Procedural - Creating clear and unambiguous instructions for a computer to follow
  2. Abstract - Finding similiarity, or generality, amongst seemingly different things (avoiding repetition)
  3. Systematic - Breaking a big problem down into smaller pieces (big-picture and decision-making) and how the pieces fit together
  4. Technological Empathy - Understanding how a computer works, what it is, and what it's good and bad at doing (understanding tools)
  5. Debugging - Systematic process of identifying the cause of a computer that doesn't work. If a program doesn't work, there is a mistake.

Here are the steps to fix the problem
1. Collect evidence (what makes the program fail?)
2. Generate theories (what caused this?)
3. Test theories (if theory is right, how do I find out?
4. Fix the problem

Adding CSS to HTML

Avoiding repetition

Avoiding mistakes and consistency:

When doing the same thing over and over it becomes easy to make a mistake. The benefit of CSS is that it allows you make changes to parts of the document with a few lines of code instead of going through and manually applying the style to each individual line. For example, with this page, by assinging each lesson and header a class, I'm able to set the style with just one bit of code. I didn't need to set it each time. Very, very useful.

What is CSS?

CSS stands for cascading style sheets. It is the code that applies the look to the HTML structure. It cascades throughout the whole document. Meaning it not only applies to element's that directly match, but element's children as well. This is inheritance. If there are multiple rules for a child, the most specific wins. You link to the CSS file in the head of the html doc. Put the CSS file and HTML file in the same folder.

Boxes, boxes, and boxes

Earlier I mentioned to see everything as boxes in HTML. This is not only helpful in developing structure but it makes it easier to style. There are 4 components to the boxes: Margin, border, padding and conent.

Because there are four components, sizing can be different. Also, browser display content differently. This will be important in the future.

Display: flex; allows divs to appear next to each other instead of stacked. Will use this to build a TOC (table of contents) that appears next to the conent. How do I make it stay static while content moves to make navigation easier?

Two techniques:

  1. use percents instead of pixels
  2. boxsizing: border-box

Code, test, refine

The strategy:

Object Oriented Programming

What is Object Oriented Programming?

OOP is a programming language model organized around objects rather than “actions” and data rather than logic. It is a design philosophy. This just means describing what things are and what they can do. It is considered more 'new school' vs the 'old school' procedural programming.

Why is OOP good for thinking about big projects?

OOP is good for thinking about large projects because of the way it is structured around identifying and using objects. This will help you have a cleaner, more organized project that can be modified easier than if you used a procedural language.

What are the advantages of OOP for large programming projects?

The advantage of OOP for large projects is that you can easily modify your code. Instead of having to write a whole new procedure if you want to tweak something you can just just change something in the class.

Another advantage is faster development because you're able to reuse code that is already written from the library.

Since I imagine writing a large program would be hard, approaching it with an OOP mindset would force you to spend more time on design which would result in a better program.

Vocab.. yes, yes?

Class - Think of this as a a blueprint, a detailed description that defines what an object will be (behavior and attributes). Once defined, we can create multiple objects from it (ex. multiple houses from the same blueprint.)

Object and instance - Structures that contain both data and procedures (the things built from blueprints).

Method - A function which is a member of a class.

Library - This is what comes with when you install it. The library has a bunch of modules (code that has already been written) which you can import into your own code and make things much faster.

Class variables - Attributes shared by all instances of the class.

Inheritance - The idea that a sub class class will get most or all of its features from a parent class (like how you inherit traits from your mom and dad). This is great form of code re-use. You can create a new class and instead of writing it from scratch, we can base it on an existing class.

Method overriding - sometimes you want the child to behave differently so you override the function in the child, replacing the functionality.

Making abstract connections

OOP is like using classes in HTML and CSS when making webpages. Instead of using inline CSS to style the page, we categorise the HTML elements on the webpage using CSS classes which specify the design properties to be applied on them. Since classes and CSS classes both act as blueprints it helps to avoid repetition.
